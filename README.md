  <h3>&nbsp;⬡ &nbsp;BenchMarkIQ AI &nbsp;—&nbsp; AI-Powered Product Performance Benchmarking Platform &nbsp; 🚀</h3>
</summary>

<br/>

<div align="center">

[![BenchMarkIQ](https://img.shields.io/badge/🌐_LIVE_DEMO-bench--mark--iq--five.vercel.app-7C3AED?style=for-the-badge)](https://bench-mark-iq-five.vercel.app)
[![GitHub](https://img.shields.io/badge/GitHub-Repository-181717?style=for-the-badge&logo=github)](https://github.com/Ashu411/Sales-Performance-Gap-Analysis-for-Preventing-Underperformance-of-Products-/tree/main)
[![Status](https://img.shields.io/badge/Status-🟢_LIVE-10B981?style=for-the-badge)]()

</div>

> **The flagship project.** An end-to-end AI analytics platform that transforms raw sales CSV datasets into benchmark-driven business intelligence — complete with product performance classification, gap detection, AI-generated recommendations, an interactive React dashboard, and one-click PDF exports. Zero manual configuration required.

<br/>

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | React · TypeScript · Tailwind CSS · FastAPI · Python · PostgreSQL · Neon · Gemini AI · Clerk · Cloudinary · Vercel · Render |
| **Scale** | Processes 10,000+ row datasets through an 8-stage automated pipeline in under 60 seconds |
| **Core Engine** | Proprietary benchmarking algorithm: identifies top performers → creates baseline → scores every SKU across 6 KPIs |
| **AI Layer** | Gemini AI generates context-aware, prioritized business recommendations tied to exact data gaps |
| **Features** | Natural language queries · Product tier classification (Top/Mid/Low) · Gap analysis · PDF reports · Live charts |
| **Impact** | Eliminates manual BI analysis; delivers actionable growth intelligence to non-technical business teams |

</div>

<br/>

```
Pipeline:  CSV Upload → Cleaning → Feature Engineering → Benchmarking → 
           Scoring → Classification → Gemini AI → Dashboard → PDF Export
```

**Engineering Architecture:** The platform implements a proprietary benchmarking engine that automatically identifies top-performing products and uses their KPI profile as the performance baseline. Every SKU in the dataset is then scored and classified into tiers across Revenue, Profit, Margin, Discount, Quantity, and Growth dimensions. The gap calculation engine produces precise delta metrics for each product. These gaps feed directly into the Gemini AI recommendation layer, which generates prioritized, context-aware corrective actions in natural language. The full system runs on a Vercel (React frontend) + Render (FastAPI backend) + Neon PostgreSQL architecture, with Clerk authentication and Cloudinary file handling.

<div align="center">

> A C++ simulation engine that models pandemic transmission dynamics across a city-graph network using probabilistic infection models — enabling comparative analysis of lockdown strategies and containment timing.

<br/>

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | C++ · Graph Theory · Probabilistic Modeling |
| **Architecture** | City-nodes with population/infection state + weighted transmission edges |
| **Simulation** | Tick-based probabilistic spread; infection propagates per contact rate and edge weight |
| **Features** | Inter-city transmission · Lockdown policy simulation · Dynamic edge weight adjustment |
| **Output** | Real-time statistics: active cases · recovery rates · containment effectiveness metrics |
| **Impact** | Analytical framework for comparing early vs. delayed containment strategy outcomes |

</div>

<br/>

**Engineering Highlights:** Cities are modeled as graph nodes with mutable infection states. Lockdown interventions reduce edge transmission weights dynamically — allowing the simulation to model partial lockdowns, full closures, and phased reopening. The tick-based engine outputs statistical snapshots at configurable intervals, enabling longitudinal comparison of intervention scenarios.

## ◈ Featured Projects

<details>
<summary><b>⬡ &nbsp;BenchMarkIQ AI — AI-Powered Product Performance Benchmarking Platform</b></summary>
<br>

> An end-to-end AI analytics platform that transforms raw sales CSV datasets into benchmark-driven business intelligence, product performance classifications, and AI-generated growth recommendations — all with zero manual configuration.

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | React · TypeScript · Tailwind CSS · FastAPI · Python · PostgreSQL · Neon · Gemini AI · Clerk · Cloudinary · Vercel · Render |
| **Scale** | Handles 10,000+ row datasets; processes full analytics pipeline in under 60 seconds |
| **Performance** | Automated 8-stage data pipeline: cleaning → feature engineering → benchmarking → scoring → classification → AI insights |
| **Intelligence** | Gemini AI powers natural language queries and context-aware, prioritized business recommendations |
| **Impact** | Identifies product performance gaps across Revenue, Profit, Margin, Discount, and Growth KPIs |
| **Repository** | [github.com/Ashu411/Sales-Performance-Gap-Analysis...](https://github.com/Ashu411/Sales-Performance-Gap-Analysis-for-Preventing-Underperformance-of-Products-/tree/main) |

</div>

**Engineering Highlights:**
The platform implements a proprietary benchmarking engine that automatically identifies top-performing products and uses them as the performance baseline. Every other product is scored and classified into tiers — Top, Mid, and Low — with precise gap metrics calculated across multiple KPIs. The AI recommendation engine leverages Gemini AI to generate context-aware, prioritized corrective actions tied directly to identified gaps. The full system is deployable on Vercel + Render with Neon serverless PostgreSQL, and Clerk handles authentication. A live React dashboard renders real-time charts, product rankings, and an interactive AI recommendation panel — all exportable as a structured PDF report.

<details>
<summary><b>⬡ &nbsp;Pandemic Spread Simulation — Graph-Based Epidemiological Modeling System</b></summary>
<br>

> A C++ simulation system that models pandemic transmission dynamics between cities using graph theory and probabilistic infection models, with support for lockdown strategy analysis and real-time containment statistics.

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | C++ · Graph Theory · Probabilistic Models |
| **Architecture** | City-node graph with weighted transmission edges; probabilistic spread per simulation tick |
| **Features** | Inter-city transmission modeling, lockdown policy simulation, real-time statistics output |
| **Scale** | Multi-city simulation with configurable population, infection rates, and intervention timing |
| **Impact** | Provides analytical framework for evaluating containment effectiveness under different policies |
| **Repository** | [github.com/Ashu411/Pandemic-Spread-Simulation](https://github.com/Ashu411/Pandemic-Spread-Simulation-and-Containment-Analysis-System) |

</div>

**Engineering Highlights:**
The simulation engine represents cities as graph nodes with population and infection state, and inter-city travel as weighted edges with transmission probability. Each simulation tick propagates infection probabilistically based on contact rates and current spread state. Lockdown interventions reduce edge weights dynamically, allowing comparative analysis of early vs. late containment strategies. The system outputs real-time statistics including active cases, recovery rates, and containment effectiveness metrics.
